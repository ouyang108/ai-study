# 技术点时效性审查（2026-05-27）

> 审查范围：当前目录下 `00` 到 `25` 的全部 Markdown 文档。  
> 结论口径：本文件只标注可能过时、需要改写或需要加注释的技术点；不代表原文全部不可用。

## 总体结论

这批文档整体仍然可作为 AI 工程入门和进阶材料使用，但有几类内容需要更新：

1. **模型与价格表最容易过期**：OpenAI、Claude、DeepSeek、Gemini 的模型名、价格、上下文长度、推荐默认模型均已有变化或存在不可靠表述。
2. **OpenAI 官方新项目可优先看 Responses API，但本系列保留 Chat Completions 作为通用示例**：DeepSeek、vLLM、Ollama、LiteLLM 等 OpenAI-compatible 服务普遍支持 `chat.completions.create()`，所以不需要为了“最新写法”强行替换全部示例。
3. **DeepSeek 模型名与价格只做快照标注**：`deepseek-chat` / `deepseek-reasoner` 是常见兼容写法，但具体模型、上下文、价格以 DeepSeek 最新文档为准。
4. **Claude 4.6 / Haiku 4.5 等模型名需要核验或替换**：Anthropic 官方模型表中没有这些 API model ID。生产示例应使用官方具体快照 ID，例如 `claude-sonnet-4-20250514`、`claude-opus-4-1-20250805`、`claude-3-5-haiku-20241022`。
5. **MCP 官方服务器列表与 SDK 示例需要更新**：MCP servers 仓库现在强调“参考实现”，并建议通过 MCP Registry 查找服务器；部分旧 `@modelcontextprotocol/server-*` 包已在 npm 标记 deprecated。
6. **`@xenova/transformers` 示例应加“旧命名空间/旧版 Transformers.js”提示**：npm 未标记 deprecated，但 Hugging Face 当前维护的包名是 `@huggingface/transformers`。

## MCP 包名补充说明

截至 2026-05-27，npm 上同时存在：

| 包名 | 当前版本 | 定位 | 备注 |
|---|---:|---|---|
| `@modelcontextprotocol/sdk` | `1.29.0` | 稳定的一体化 TypeScript SDK | 服务端通常从 `@modelcontextprotocol/sdk/server/mcp.js` 引入 `McpServer`，并用 `server.tool(...)` 注册工具。 |
| `@modelcontextprotocol/server` | `2.0.0-alpha.2` | 2.0 alpha 的服务端拆包 | 来自同一个 `modelcontextprotocol/typescript-sdk` 仓库，依赖 `zod@^4.0`；官方新示例使用 `server.registerTool(...)` 和 `zod/v4`。 |
| `@modelcontextprotocol/client` | `2.0.0-alpha.2` | 2.0 alpha 的客户端拆包 | 同上，属于 alpha 版本线。 |

因此，`02-MCP协议-Model-Context-Protocol.md` 后续改写时不要简单说“没有 `@modelcontextprotocol/server`”。更准确的说法是：

- 稳定教材：使用 `npm install @modelcontextprotocol/sdk zod`，工具 API 写 `server.tool(...)`。
- 前沿/alpha 教材：可以介绍 `@modelcontextprotocol/server`，工具 API 写 `server.registerTool(...)`，并注明它当前是 `2.0.0-alpha.2`。
- 工具 schema：无论稳定线还是 alpha 线，都应突出 `McpServer + Zod schema` 这种写法。
